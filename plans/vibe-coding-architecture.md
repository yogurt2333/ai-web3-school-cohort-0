## 一、架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        你的笔记本                                │
│  ┌────────────────────┐    ┌──────────────────────────────────┐ │
│  │ VSCode (Remote SSH) │    │ 微信                            │ │
│  │ 直接看服务器文件    │    │ 与我 (Hermes) 对话              │ │
│  └────────┬───────────┘    └────────┬─────────────────────────┘ │
└───────────┼─────────────────────────┼───────────────────────────┘
            │ SSH                     │ WeChat
            ▼                         ▼
┌─────────────────────────────────────────────────────────────────┐
│                        服务器                                    │
│                                                                  │
│  ┌──────────┐     ┌────────────┐     ┌──────────────────────┐  │
│  │ Hermes   │────→│ tmux 会话  │────→│ Code Agent            │  │
│  │ (DeepSeek│     │ 后台持久化  │     │ (OpenCode/Claude...)  │  │
│  │  调度层) │     │            │     │                      │  │
│  │          │     │ 你随时     │     │ • 读写代码            │  │
│  │ • 规划   │     │ attach 看  │     │ • 跑测试              │  │
│  │ • 答疑   │     │ 一手输出   │     │ • git commit          │  │
│  │ • 知识库 │     │            │     │                      │  │
│  └──────────┘     └────────────┘     └──────────────────────┘  │
│                        ↑                          ↑             │
│                  你直接看 TUI              我监控 log            │
└─────────────────────────────────────────────────────────────────┘
```

### 核心原则

| 层 | 谁 | 模型 | 干什么 | 不干什么 |
|----|----|------|--------|---------|
| **调度层** | Hermes（我） | DeepSeek（便宜） | 规划、答疑、启动 agent、总结 | 不润色 code agent 的输入输出 |
| **执行层** | Code Agent | 强模型（贵） | 写代码、改文件、跑命令 | 不负责顶层设计决策 |
| **展示层** | VSCode + tmux | 无 | 显示一手代码和输出 | 无 AI 介入 |

## 二、关键设计决策

### 决策 1：Code Agent 选型

| 选项 | 模型 | 国内网络 | 成本 | 代码质量 | 安装复杂度 |
|------|------|---------|------|---------|-----------|
| **A) OpenCode + DeepSeek v4 Pro** | DeepSeek | ✅ 直连 | 低 (≈¥0.5/百万token) | ⭐⭐⭐⭐ | 低 |
| B) OpenCode + Claude Sonnet (ccswitch) | Claude | ⚠️ 需中转 | 高 ($3/百万token) | ⭐⭐⭐⭐⭐ | 中 |
| C) Codex (OpenAI) | GPT-4o | ⚠️ 需代理 | 中 ($2.5/百万token) | ⭐⭐⭐⭐ | 低 |
| D) Claude Code | Sonnet/Opus | ⚠️ 需中转 | 高 | ⭐⭐⭐⭐⭐ | 中 |

**初步倾向**: 方案 A，零摩擦起步。后续可无缝升级到 B（OpenCode 模型无关）。

### 决策 2：代码如何同步

| 方式 | 体验 | 需要额外配置 |
|------|------|------------|
| **A) VSCode Remote SSH** | ✅ 就像本地文件，零推拉 | SSH 免密登录 |
| B) GitHub 推拉 | ❌ 改→推→拉→改，来回折腾 | GitHub 认证 |
| C) Syncthing / rsync | ⚠️ 实时双向同步，但偶尔冲突 | 额外工具 |

**推荐**: A（VSCode Remote SSH），零摩擦。

### 决策 3：对话模式（信息流）

```
        我管规划方向                     你管执行查看
        ──────────                      ──────────

[方向讨论]                              [代码查看]
 你: "我想写个 XXX"                       你: VSCode SSH 打开文件
 我: "好思路，先拆成三步..."              你: tmux attach 看 OpenCode TUI
                                         你: 看到原始代码 diff / 编译报错
[我启动 agent]
 我: 创建 tmux 会话                      [你给 agent 指令]
 我: 启动 OpenCode                       你: 直接在 TUI 里打字
 我: 把上下文喂给它                      你: 看它实时输出
                                        你: Ctrl+B D 断开

[我总结+下一步]                          [你回来汇报]
 我: "它跑了啥，改了啥文件"              你: "搞完了，VSCode 上能看到"
 我: "下一步建议是..."
```

**关键**: 我不当传话筒。我只在 **规划阶段** 和 **收尾总结** 介入，执行阶段你直接跟 agent 对话。

## 三、成本估算

### 场景：写完一个简单的 Web3 交互页面

| 环节 | 模型 | Token 消耗（估） | 费用 |
|------|------|----------------|------|
| 你跟我讨论需求（10 轮） | DeepSeek v4 | ~50K in / ~10K out | ≈ ¥0.03 |
| OpenCode 写代码 | DeepSeek v4 Pro | ~200K in / ~50K out | ≈ ¥0.12 |
| 我帮你总结 | DeepSeek v4 | ~30K in / ~5K out | ≈ ¥0.02 |
| **合计** | | | **≈ ¥0.17** |

如果换 Claude Sonnet，同样工作量 ≈ **$0.6 (~¥4.3)**，约 25 倍差价。

### 结论

用 DeepSeek v4 Pro 做执行层，按 vibe coding 强度（每天 2-3 小时），**月花费大概 ¥10-30**，完全可以接受。

## 五、风险与缓解

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| tmux 会话被误杀 | 低 | 中 | 配置 tmux 自动保存日志 |
| DeepSeek 写代码质量不够 | 中 | 中 | OpenCode 无缝切后端到 Claude |
| SSH 断连后 tmux 还在 | — | — | tmux 的特性就是断线不丢 ✅ |
| 我忍不住润色 | 中 | 低 | 定好规矩：执行阶段我不插嘴 |
| 你觉得三个人太乱 | 低 | 高 | 随时可以退回「你↔我」两人模式 |
