# 基础概念说明：LLM → Workflow → Agent → 钱包 → 签名 → 交易 → 合约

> AI × Web3 School Cohort 0 — Week 1 交付物
> 一条链打通 AI 与 Web3 的核心概念

---

## 一、总览：两条线，一个交汇

```
AI 线：  LLM →  Prompt →  Workflow  →  Agent
                                      ↓
                                     交叉实验
                                      ↑
Web3 线：钱包 → 私钥 → 签名 → 交易 → 合约
```

**核心认知：AI 是生成者，Web3 是执行场。AI 生成意图，Web3 保证执行不被篡改。**

---

## 二、AI 线：从概率到行动

### 1. LLM（大语言模型）

**是什么**：一个读过半个互联网的「高级续写机器」。输入一段文字，它预测最可能的下一个字/词，逐个生成完整回答。

**关键认知**：
- LLM **没有意图，只有概率**。它不是在"思考"，而是在"猜你最想听什么"
- 上下文窗口 = 短时记忆。Claude 有 200K ≈ 15 万字，但注意力会随长度衰减
- 擅长：总结、改写、生成、翻译、解释、头脑风暴
- 不擅长：精确计算、长链推理、实时信息、确定性操作

**前端类比**：LLM 像 Vue 的 `computed`——你给它依赖（prompt），它每次重新计算输出。但 computed 是确定性的，LLM 每次可能给你不同的结果。

### 2. Prompt（提示词）

**是什么**：给 LLM 的输入——指令、上下文、期望格式的组合。

**关键认知**：
- Prompt 是**唯一的人机接口**，质量直接决定输出
- 好 prompt = 明确角色 + 具体任务 + 输出格式 + 示例
- Prompt 越模糊，LLM 的自由度越大，你越不可控

```
不好的 prompt：帮我写个合约
好的 prompt：你是一个 Solidity 专家。写一个 Counter 合约，功能是 increment、
           decrement 和 getCount。使用 SPDX 许可证，Solidity ^0.8.0。
           输出完整代码 + 解释每段作用。
```

### 3. Workflow（工作流）

**是什么**：把多个 LLM 调用串联起来，前一步的输出作为后一步的输入。

**关键认知**：
- 单次 LLM 调用 ≈ 一个函数，workflow ≈ 函数链
- Workflow 是确定的——你写死每一步的逻辑
- 典型模式：输入 → LLM 处理 → 检查 → LLM 优化 → 输出

**前端类比**：workflow 像 `Promise.then().then().catch()` 链——每一步都知道下一步做什么。

### 4. Agent（智能体）

**是什么**：LLM + 工具 + 记忆 + 自主决策。Workflow 是"按剧本演"，Agent 是"自己写剧本"。

**关键认知**：
- Agent 能自己决定：用什么工具、什么时候调用、怎么处理结果
- 核心组件：LLM（大脑）+ Tool（手脚）+ Memory（记忆）+ Loop（持续执行）
- 风险：Agent 越自由，失控边界越大。必须加 **Guardrails（护栏）**

**AI 线一句话总结**：
> LLM 是大脑，Prompt 是语言，Workflow 是剧本，Agent 是自己写剧本的演员。

## 三、Web3 线：从身份到契约

### 5. 钱包

**是什么**：管理你链上身份和资产的工具。不是"存钱的地方"，是"管钥匙的地方"。

**关键认知**：
- 钱包 ≈ 钥匙串。你的资产在链上（公开账本），钱包只保管能操作它们的钥匙
- 钱包生成的是一对密码学钥匙：**私钥（秘密）→ 公钥 → 地址（公开身份）**
- 地址 = 公钥哈希后的一串字符（`0x...`），公开给任何人
- 助记词（12/24 个英文单词）= 私钥的人类友好版，**谁有谁拥有钱包**
- **失去助记词 = 永久失去钱包**，没有"找回密码"按钮

**前端类比**：钱包就像浏览器的 `localStorage`——存的是密钥（私钥），不是内容本身。页面刷新（换设备）了，localStorage 里的密钥没了，但链上数据还在。

### 6. 签名（Sign）

**是什么**：用私钥给一段消息盖章，证明"这条消息是我发的，没被改过"。

**关键认知**：
- 签名 ≠ 交易。签名是**预处理**，交易是**上链执行**
- 签名是密码学魔法：用私钥加密一段数据，任何人都可以用你的公钥验证"这确实是你签的"
- 签名的过程发生在**本地**（钱包里），不上链、不花钱
- 你平时"授权 dApp 登录"就是在签名一条消息

### 7. 交易（Transaction）

**是什么**：签名后广播到区块链网络，等待被打包进区块的执行指令。

**关键认知**：
- 交易 = 签名后的消息 + 发送到网络
- 状态变化线：组报文 → 钱包弹窗让你检查 → 你签名 → 广播 → 待确认 → 打包上链
- **每一步你都可以取消**（关掉钱包弹窗 = 交易从未存在）
- 但你一旦点了"确认"并签名，交易就**不可逆转**
- Gas = 你为这次计算付的手续费
- Nonce = 交易序号（从 0 开始，每发一笔 +1），防止双花

**前端类比**：签名 ≈ 在本地 `JSON.stringify()` + 数字签名，交易 ≈ `fetch('/api/broadcast', { method: 'POST', body: signedTx })`。

### 8. 智能合约（Smart Contract）

**是什么**：部署在链上的程序，一旦部署就**不可篡改**，按代码逻辑自动执行。

**关键认知**：
- 合约就是**链上的后端**，不需要你维护服务器
- 普通后端：代码可以随时热更新，数据库可以回滚
- 智能合约：代码部署后**改不了**，bug 就是永久漏洞（史上最大的教训：The DAO 黑客事件）
- 调用合约也是发交易，同样要 Gas
- 合约部署也是发交易（to 地址为空，input 为合约字节码）

**最形象的对比：**

| | 普通后端 | 智能合约 |
|--|---------|---------|
| 部署方式 | SSH 上传文件 / CI/CD | 发一笔交易上链 |
| 更新 | 随时改代码 | ❌ 不可改（除非设计代理模式） |
| 停机 | 服务器挂了就挂了 | ✅ 永远在线（只要链在跑） |
| 用户认证 | JWT / Session | 钱包签名 |
| 手续费 | 开发者付服务器钱 | 用户付 Gas |
| 数据存储 | 数据库，可以改 | 链上状态，永久记录 |

## 四、AI × Web3 交汇：最小交叉实验

### 交汇逻辑

```
AI 生成 → 人工复核 → 钱包签名 → 链上执行 → 区块浏览器验证
```

**为什么需要人参与**：
1. LLM 可能生成有 bug 的合约代码
2. LLM 可能推荐错误的 Gas 参数
3. LLM 可能误解你的意图，生成危险操作（转账到错误地址）
4. **AI 可以加速，但最终签名权在人的钱包里**

### 现实世界怎么做

1. AI 帮你写合约代码 / 交易脚本
2. **你读懂并审查** AI 的输出
3. 钱包弹窗让你再次确认交易内容
4. 确认无误 → 签名 → 上链
5. 区块浏览器二次验证交易结果

**这就是 Web3 的安全铁三角**：
> AI 生成 → 人工复核 → 钱包确认

每一层都是安全冗余，少一层就是风险敞口。

## 五、一些冷水和思考

- **AI 可以被引导犯错**：prompt injection 可以骗 agent 转走你的资产
- **Web3 的 UX 远不如 Web2**：丢私钥 = 丢钱，这个设计代价普通用户承受不了
- **智能合约不是银弹**：80% 的 DeFi 黑客来自合约代码 bug，AI 写代码只会加速这个数字
- **Agent 的信任问题**：你会让一个 AI agent 持有你的私钥吗？如果会，出事了找谁？

## 六、概念速查表

| 概念 | 一句话 | 类比 |
|------|--------|------|
| LLM | 概率续写机 | computed 属性 |
| Prompt | 给 AI 的指令 | 函数参数 |
| Workflow | 固定步骤链 | Promise 链 |
| Agent | 自主决策体 | 自己写剧本的演员 |
| 钱包 | 管钥匙的工具 | 钥匙串 |
| 私钥 | 账号的终极密码 | 银行 U 盾 |
| 签名 | 盖章证明"是我发的" | 数字手写签名 |
| 交易 | 签名后广播上链 | POST 请求 |
| Gas | 计算资源费 | AWS 按量计费 |
| 合约 | 不可改的链上程序 | 永久运行的后端 |
